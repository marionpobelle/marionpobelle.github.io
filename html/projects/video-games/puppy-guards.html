OVERVIEW

Puppy Guards is a 2 - 4 players cooperative couch game in which players take on the role of museum watchmen in charge of protecting the art pieces from at best clumsy, at worst malicious visitors!

Build barricades with furniture, bark at the visitors or knock them down. Everything is worth it to save the relics from the disaster.

As the only programmer on Puppy Guards, I had to undertake every aspect of programming the game, as well as assess the feasibility of each feature within the time and means we had. My duties included:

  • Develop gameplay features (player controller, moving geometry…)
  • Develop and refine AI behavior (state machine, illusion of credible behavior…)
  • Implement User Interface (design menus, set up navigation…)
  • Asset integration (User Interface, 3D models, animations…)
  • Improve the game designer’s work environment with editor programming and a custom inspector
  • Performance control and optimization
  • Collaborating closely with environment artists, sound designers and game designers
  • Maintain a feature list with regular updates so that everyone can track progress
  • Writing technical documentation (naming convention, integration procedure…)
  • And more!

As the primary point of contact for the rest of the team, I created and maintained the game’s feature list and documentation. After that, I detailed the structure of each feature on paper, prototyped them, processed the team’s feedback as I went, and then worked on and owned them until polished.

Some of my teammates wanted to learn programming, so I did my best to help them, be available and allow them to implement small features themselves.

GAMEPLAY

Puppy Guards is a gamepad only game as it was designed as a couch coop. Please note that any input encountered refers to an Xbox controller.

The player’s movement uses Unity’s physics system for a fun controller and a goofy look. To move the player, we used the velocity of its Rigidbody as well as a speed-up rate and a slow down factor to fine-tune the movement.

The three principal player actions are Dash (B), Bark (X) and Grab (A).

  • The Dash (B) input improves the players’ movement, but can also be used to interact with AI.
  • The Bark (X) input is used to interact with AI; its use will be detailed further below.
  • The Grab(A) input is used to grab specific geometry (furniture) and move it around.

Grabbing creates a FixedJoint between the piece of furniture and the player. We added angle control to ensure the player would be attached to the object correctly. The FixedJoint between a movable element and a player breaks when the player presses the Grab (A) input again.

AI

Puppy Guards resembles Tower Defense games, except the “enemies” are not really targeting the structures. Because of that, the artificial intelligence (AI) had to give the illusion of credible behavior. It had to be simple, but look like it had specific objectives for people to attribute intentions to it. 

Every AI type inherits from a base State Machine. The states available to an AI are Idle, Move and Special. The Idle state gives the illusion that an AI stops to take in the environment and allow players to breathe. The Move state allows each AI to move from point A to point B, point A being its current position and point B being selected randomly within a grid of possible positions. The Special state allows each AI to execute a unique special behavior if it has one.

Each type of AI has a special behavior. It makes the game more fun, varied and unpredictable! I implemented three different types:

  • Sheep: a basic AI. Its special behavior is that when a player barks within its range, a sheep will walk away in the opposite direction. They also get stunned when dashed into. This allows for easy crowd control.
  • Goat: capable of pushing furniture on contact. This special behavior was designed to disturb the players’ plan with furniture positioning. To control this AI, the player can simply tap it to make it change destination. They will not react to barks or dashes.
  • Chicks: act as excited children and will randomly start to run, stunning players on contact. This special behavior forces players to rely on quick reflexes and good terrain coverage. This AI does not care if you bark at it, but its small size will allow players to propel it out of the way with a dash.

LOCAL MULTIPLAYER

It was my first time implementing local multiplayer so this feature is the one that took me the most amount of time. I used the new input system as well as the C# class generated by the player controls. I made the system scalable and easily reusable throughout projects.
